# edit-snapshot 使用简介

这是一个给文件修改流程加保护的 skill。

它适合在要改代码、改配置、批量替换、生成文件这类任务开始前使用。它的作用不是管理完整的 Git 工作流，而是在真正动文件之前先留一个恢复点，改完以后再把结果和回退方式清楚告诉你。

## 它能帮你做什么

当任务要修改文件时，它会先记录一个修改前恢复点。

修改完成后，它会。

- 在有需要时保存一个修改后的结果快照
- 显示最近几次 Git 提交记录
- 告诉你这次修改支持回退
- 提示你常见的回退命令

## 你会得到什么信息

任务结束后，你通常会知道这些内容。

- 修改前的快照是什么
- 修改后有没有生成新的提交
- 最近几次提交记录是什么
- 想撤回这次改动时应该看哪些命令

## 常用命令

```bash
{baseDir}/scripts/helper.sh pre "本次修改原因"
# 这里执行实际修改
{baseDir}/scripts/helper.sh post "本次修改原因"
{baseDir}/scripts/helper.sh recent 5
{baseDir}/scripts/helper.sh rollback-help
```

## 什么时候适合用

这些情况建议用。

- 改代码
- 改配置
- 自动生成文件
- 批量替换
- 重构
- 运行会写文件的命令

这些情况一般不用。

- 只读查看
- 搜索
- 解释代码
- 只做分析

## 需要知道的边界

这个 skill 主要是在当前目录的本地 Git 仓库里工作。

它的重点是给修改过程加保护，而不是接管完整的 Git 协作流程。所以它不会替您自动做完整的版本管理决策，只会把恢复点和回退线索准备好。

另外，如果任务开始前仓库本来就已经有未提交改动，那么修改前快照会把这些现有改动一起保存进去。这么做是为了先保护现场。

还有一点也值得注意。未被 `.gitignore` 忽略的新增文件、生成物、二进制文件，甚至误放进工作区的敏感文件，也可能一起被保存到快照里。

## 安装后怎么确认可用

```bash
openclaw skills list
openclaw skills info edit-snapshot
openclaw skills check
```

## 回退时常见会看到的命令

```bash
git revert <post_sha>
git reset --hard <pre_sha>
git reflog -n 10
```

这些命令分别对应。

- 撤销本次结果提交
- 直接回到修改前
- 查看更多可恢复位置
